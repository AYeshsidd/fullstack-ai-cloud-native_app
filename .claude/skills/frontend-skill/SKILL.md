# Frontend Layer (Next.js)

## Instructions

1. **Pages & Routing**
   - Use Next.js App Router
   - Build complete frontend pages for a full-stack Todo application,
     including authentication, dashboard/tasks list, task details, and layouts
   - Protect routes that require authentication

2. **Layout Structure**
   - Use shared root layout
   - Consistent header, sidebar, and navigation
   - Responsive page structure for dashboard-style apps

3. **Components**
   - Build reusable UI components
   - Separate presentational and interactive components
   - Use client components only when required

4. **UI Styling & Theme**
   - Style using Tailwind CSS
   - Mobile-first responsive design
   - Implement light and dark mode
   - Support system preference and manual theme toggle
   - Consistent spacing, typography, and colors

5. **Icons Usage**
   - Use a consistent icon library (e.g. Lucide or Heroicons)
   - Add icons to navigation, buttons, actions, and empty states
   - Use icons to improve UX clarity, not decoration only
   - Ensure icons adapt to light and dark mode

6. **API Integration**
   - Centralize backend calls in API client
   - Attach JWT token to all requests
   - Handle loading and error states

## Best Practices
- Default to server components
- Keep components small and reusable
- Avoid inline styles
- Handle empty and error states gracefully
- Ensure accessibility basics (labels, focus, contrast, aria where needed)
